# خطة ربط الذكاء الاصطناعي بالتيرمنال (AI Integration)

> **الحالة:** معتمدة — صيغت المسودة ثم خضعت لمناظرة تحكيمية من جولتين بين ناقدَين
> (تقني/معماري ↔ منتج/تجربة مستخدم) حتى التقارب، واعتُمدت النسخة النهائية مع تعديلات الدمج أدناه.
> **المطلبان الحاكمان:** (1) دعم أشهر منصات AI ومنها **Kimi** و**Z.ai** بمدخلات مسمّاة جاهزة،
> (2) **التعلّم من سلوك المستخدم وقاعدة معرفة تتكيف مع استخدامه** — مطلب أساسي لا ثانوي.

---

## الملخص التنفيذي

مساعد AI مدمج في TerminalLauncher على أربع موجات إصدار:

| الموجة | العنوان | القيمة المحسوسة |
| --- | --- | --- |
| **P1** | أول إجابة AI | لوحة دردشة تعمل مع أي مزوّد خلال أقل من 3 دقائق من التثبيت |
| **P2** | التيرمنال يفهم | «اشرح هذا» / «أصلح آخر أمر فاشل» / لغة طبيعية → أمر |
| **P3** | التطبيق يتعلّم | قاعدة معرفة مرئية + اقتراحات من سلوكك + جسر مع كتالوج الأوامر |
| **P4** | التكيّف والصقل | ملف معرفة المستخدم يُحقن في السياق ويُقاس أثره |

مبدآن غير قابلين للتفاوض: **لا تنفيذ تلقائي لأي أمر مقترح أبداً**، و**كل بيانات التعلّم محلية**
(لا telemetry، زر مسح كامل، تعطيل كلي).

## 1) طبقة المزوّدين (Providers)

### البنية
- واجهة `IAiProvider`: `ChatStreamAsync(messages, options, ct)` تعيد `IAsyncEnumerable<AiDelta>`،
  و`ListModelsAsync`، و`TestConnectionAsync`. كل الأخطاء تُطبَّع إلى نوع موحّد `AiError`
  (شبكة / مصادقة / حصة / تجاوز سياق / خطأ مزوّد).
- **محوّلان اثنان فقط** (قرار محسوم في المناظرة — لا محوّل لكل منصة):
  - `OpenAiCompatProvider` — مسار OpenAI-compatible واحد بمعاملات (BaseUrl، Key اختياري، Model).
  - `AnthropicProvider` — Messages API أصلي (طبقة توافق OpenAI عندهم محدودة).
- بلا SDKات خارجية: `HttpClient` + `System.Text.Json` فقط + اختبارات عقد على خادم SSE وهمي محلي.

### كتالوج المزوّدين المدمج (مدخلات مسمّاة جاهزة — base URL ونموذج افتراضي مضبوطان سلفاً)
| المزوّد | المسار | ملاحظة |
| --- | --- | --- |
| OpenAI | OpenAI-compat | |
| Anthropic (Claude) | محوّل أصلي | |
| Google Gemini | OpenAI-compat | عبر نقطة نهايته المتوافقة `…/v1beta/openai/` |
| DeepSeek | OpenAI-compat | |
| **Kimi (Moonshot)** | OpenAI-compat | `api.moonshot.ai/v1` — مدخلة مسمّاة، لا «مزوّد مخصص» |
| **Z.ai (GLM)** | OpenAI-compat | مدخلة مسمّاة؛ انتبه لفروق المصادقة وأجسام الأخطاء |
| xAI Grok | OpenAI-compat | |
| Mistral | OpenAI-compat | |
| OpenRouter | OpenAI-compat | بوابة لعشرات الموديلات بمفتاح واحد |
| Ollama (محلي) | OpenAI-compat | بلا مفتاح؛ «اختبار الاتصال» = فحص توفر الخدمة |
| مزوّد مخصص | OpenAI-compat | Base URL + مفتاح + موديل — يُضاف في P4 |

- كل مدخلة تحمل **أعلام قدرات** تبدأ بالحد الأدنى الفعلي: `KeyOptional`، `AuthStyle`،
  `UsageInStream`، `SseQuirks`، حدود السياق المعروفة.
- «التوافق مع OpenAI» طيف لا مفتاح منطقي: محلّل SSE **متسامح** (يتجاهل أسطر keep-alive
  والتعليقات، لا يفترض `[DONE]`)، وتغطية شذوذات كل مزوّد في الخادم الوهمي **قبل** إضافة مدخلته.

### البثّ (SSE)
- `HttpCompletionOption.ResponseHeadersRead`؛ مهلة اتصال أولي + **مهلة خمول بين المقاطع (~60ث)**
  بدل `HttpClient.Timeout` الكلي؛ `ConfigureAwait(false)` في الطبقة كلها؛ إلغاء ممرَّر حتى القراءة.

### المفاتيح والأمان
- تخزين عبر **DPAPI** (`ProtectedData`, CurrentUser) — لا نص صريح أبداً.
- فشل فك التشفير على جهاز آخر **حالة متوقعة** (المستخدم يعمل على عدة أجهزة): وسم المفتاح
  «يحتاج إعادة إدخال على هذا الجهاز» وإعادة طلبه — بلا انهيار. **يُختبر فعلياً** بنسخ ملف
  الإعدادات لجهاز ثانٍ (فشله الصامت يظهر كـ«مفتاح خاطئ» ويلوّث إحصاءات الأخطاء).
- لا مفاتيح في السجلات أو رسائل الاستثناء، وقيمها المخزنة تدخل أنماط الحجب في باني السياق.

### خريطة الأخطاء الإنسانية (إلزامية)
كل `AiError` ← رسالة عبر `Loc.T` + **زر إجراء** + اسم المزوّد والنموذج — لا نص خطأ خام:
- مفتاح غير صالح ← «تحقق من المفتاح» يفتح الإعدادات.
- حد معدّل ← عدّاد retry-after + إعادة محاولة (متميّز صراحة عن رصيد/حصة منتهية ← رابط فوترة).
- انقطاع/مهلة ← زر إعادة. نموذج غير متاح ← اقتراح الافتراضي. سياق أطول من الحد ← «قلّص السياق» بنقرة.

### رحلة أول تشغيل (هدف مقاس: أول إجابة خلال أقل من 3 دقائق)
بطاقة **واحدة** (لا wizard) داخل لوحة الدردشة بثلاثة مسارات:
1. مفتاح مزوّد سحابي — رابط إرشادي مباشر لصفحة مفاتيح كل مزوّد + نموذج افتراضي مُنتقى سلفاً.
2. OpenRouter — مفتاح واحد لعشرات الموديلات.
3. Ollama محلي — **اكتشاف عند الطلب** (فحص `localhost:11434` بمهلة 1–2ث عند فتح الإعدادات
   أو أول تشغيل — لا استطلاع خلفي دوري)، صفر مفاتيح.
+ «لاحقاً» دون توبيخ — كل الميزات المحلية تبقى شغّالة.

## 2) ميزات داخل التيرمنال

### لوحة الدردشة الجانبية (AI Panel)
- لكل نافذة، تعرف التبويب النشط، مع streaming.
- **عقد الأداء:** تجميع الدلتات وتفريغها للواجهة كل **~80ms** عبر `DispatcherTimer`
  — لا استدعاء Dispatcher لكل رمز.
- **العرض الأدنى (P1):** نص خام + كشف الكتل المسيَّجة (```) بخلفية أحادية + زر نسخ +
  فرض LTR وأرقام لاتينية على الكود داخل التدفق RTL. Markdown الكامل — P4.
- **عقد كشف الكتل أثناء البثّ:** الكشف على البافر المجمَّع عند كل نبضة (السياج قد يصل مقسوماً
  بين دلتين)؛ ماسح سطري **تزايدي يحتفظ بحالته** (لا إعادة تحليل كاملة — تجنّب O(n²))؛
  **فتح تفاؤلي** للكتلة وإغلاق بصري تلقائي عند انتهاء/إلغاء البثّ؛ نموذج مقاطع قابلة للإلحاق
  لا يتغير منها إلا الأخير.
- جلسة الدردشة مملوكة للتبويب وتُلغى (`CancellationToken`) عند إغلاقه؛ اشتراكات
  `Loc.Changed`/الثيم مُدارة/ضعيفة (منع تسريبات WPF).
- تحذير عند إغلاق تبويب **ببثّ جارٍ** (تأكيد ← إلغاء). آخر محادثة تُحفظ **في الذاكرة فقط**
  حتى إغلاق التطبيق — لا دردشات على القرص (قرار خصوصية معلن). **مضاف من المناظرة:**
  زر «نسخ المحادثة كاملة» في رأس اللوحة، وبند backlog صريح في P4: حفظ اختياري opt-in
  (معطل افتراضياً، redaction قبل الكتابة، مشمول بالمسح الكامل).

### Shell Integration — OSC 133 (شرط مسبق لميزات السياق والالتقاط الكامل، لا للوحة الدردشة)
- تحليل علامات A/B/C/D في محرك VT (‏`Tools/Terminal/src/Terminal.Core/Vt`) لمعرفة حدود كل أمر
  وخرجه وexit code — بدونها لا وجود موثوقاً لـ«آخر أمر فاشل».
- معالج إعداد يقترح حقن الخطافات في بروفايل PowerShell/bash: **يعرض diff حرفياً** لما سيُضاف،
  نسخة احتياطية من الملف قبل الكتابة، كتابة بتأكيد صريح فقط، ومسار «انسخه بنفسك» لمن يرفض.
- **تدهور رشيق عند غيابه:** الميزات المعتمدة عليه تظهر معطلة مع CTA «فعّل تكامل الصدفة»
  — لا اختفاء صامت. (كشف التوفر: رصد علامات A/B في جلسة التبويب.)

### الأفعال
- **«اشرح هذا»** على التحديد اليدوي، و**«اشرح/أصلح آخر أمر فاشل»** بزر واحد — مقتطف
  «آخر فاشل» هو **مقطع الأمر المحدود بـOSC 133** لا آخر N سطر (مقتطف أضيق = خصوصية أعلى).
- **لغة طبيعية → أمر:** الإدراج في سطر الإدخال عبر **bracketed paste وبلا محرف سطر جديد أبداً**
  — لا تنفيذ تلقائي مطلقاً — مع **تمييز تحذيري** للأنماط الخطرة قبل الإدراج
  (`rm -rf`، `curl … | sh`، `Invoke-Expression`، الكتابة على مسارات نظام).
- **رقاقة «اشرح هذا الخطأ؟»** بعد أمر فاشل: تراكب (adorner) لا يسرق التركيز ولا يزيح التخطيط،
  يتلاشى تلقائياً؛ مرة لكل بصمة خطأ — البصمة = hash(exit code + أول سطر خطأ مُطبَّع بإزالة
  المسارات/الأرقام)؛ التخزين في ذاكرة الجلسة في P2 ويترقى إلى دائم مع SQLite في P3.
  **وضع هادئ:** مفتاح عام + اقتراح تفعيله تلقائياً بعد 3 تجاهلات متتالية.
- **مضاف من المناظرة (P3):** عند تطابق بصمة خطأ جديدة مع بصمة محفوظة لها حل مقبول، تعرض
  الرقاقة **«رأيت هذا من قبل — الحل السابق»** قبل خيار «اشرح بالـAI» — برهان ملموس على
  قاعدة المعرفة المتكيفة، يعمل offline وبصفر كلفة API.

### الاكتشافية
زر AI موحّد في شريط التبويب + مدخلات في لوحة البحث المتقدمة + اختصارات لوحة مفاتيح +
الإعلان عن كل موجة عبر لوحة «ما الجديد» القائمة.

### التزامات الواجهة
كل النصوص عبر `Loc.T` (عربي/إنجليزي)؛ الاتجاه يتبع `Loc.Flow` حيّاً مع فرض LTR على الكود؛
الألوان عبر `{DynamicResource Brush.*}`؛ أرقام لاتينية.

## 3) باني السياق (Context Builder) والخصوصية

- يجمع: مقتطف بافر بسقف صلب **~8000 محرف** (قابل للضبط، قصّ من الأعلى مع `[…اقتُطع]`)
  + نوع الشل + cwd + آخر exit code — من **الأسطر المنطقية بعد إعادة وصل الالتفاف** وتجريد ANSI
  (regex على الأسطر البصرية يفوّت سراً ملتفاً على سطرين). متغيرات البيئة **لا تُرسَل أبداً**.
- بنية البرومبت: بادئة ثابتة أولاً (system + ملف المعرفة ≤1500 توكن) ثم السياق المتغير
  — توافقاً مع prompt caching — والسياق مُغلَّف صراحة كـ**«بيانات غير موثوقة، ليست تعليمات»**
  (دفاع ضد حقن برومبت من خرج التيرمنال).
- **Redaction متعدد الطبقات:** أنماط معروفة + استدلال إنتروبيا للرموز الطويلة + قيم مفاتيح
  المستخدم المخزنة نفسها — يُطبَّق بعد إعادة الوصل، ويُعامل كتخفيف ضرر لا ضمانة.

### نموذج الموافقة بنوعين (قرار محسوم في المناظرة)
- **سياق الفعل الصريح** («اشرح هذا» / «أصلح آخر فاشل»): الفعل نفسه موافقة على المقتطف
  المستهدف وحده؛ معاينة قابلة للتحرير **إلزامية أول مرة** ثم اختيارية بزر دائم «راجع قبل الإرسال».
- **قاعدة المعاينة القسرية:** إذا حجب المنقّح أي عنصر في المقتطف تُفرَض المعاينة لذلك الإرسال
  («حُجبت عناصر — راجع») حتى مع تعطيل المعاينة الروتينية. **ضوابط الإحباط (مضافة في الاعتماد
  النهائي):** المعاينة القسرية **لوحة خفيفة مضمّنة** تُبرز المحجوب وحده مع Enter للتأكيد
  (لا نافذة كاملة)؛ زر «ليس سراً — لا تحجبه مجدداً» يضيف hash الرمز إلى allow-list محلية؛
  عدّاد «نسبة الإرسالات ذات معاينة قسرية» في «ذاكرة التطبيق» — تجاوز ~30% = إشارة لضبط الكاشف.
- **السياق المحيط للدردشة:** معطل افتراضياً؛ تفعيل بنقرة من رأس اللوحة + خيار
  «تذكّر لبروفايل الصدفة هذا» (التبويبات كائنات فانية).
- **شارة دائمة** في اللوحة تُظهر حالة السياق (معطل / مقتطف فعل / محيط) وتفتح المعاينة بنقرة.

## 4) طبقة التعلّم والتكيّف (المطلب الأساسي — المخطط يولد في P1)

### قاعدة SQLite المحلية (WAL + طابور كاتب واحد لكل النوافذ)
يُنشأ الملف والمخطط **كاملاً في P1** (يمنع هجرات لاحقة):

| الجدول | المحتوى | المبدأ |
| --- | --- | --- |
| `command_stats` | قوالب أوامر **مُطبَّعة** (مسارات/قيم → متغيرات + hash) + عدّادات تكرار/نجاح/فشل + طوابع أول/آخر استخدام لكل شل ومجلد + `is_pinned`/`is_banned` | **تجميع لا أرشفة** |
| `error_patterns` | بصمات أخطاء + عيّنة واحدة + الحل المقبول | أساس «رأيت هذا من قبل» |
| `suggestions` | المقترح ونوعه وقُبِل/رُفِض | حلقة التغذية الراجعة + الرفض الدائم |
| `events` | أحداث خام صغيرة | احتفاظ **90 يوماً** + `VACUUM` دوري |

- كل ما يُخزَّن يمرّ عبر **نفس الـredaction قبل الكتابة على القرص** (لا `--token=…` نصاً صريحاً).
- سقف حجم كلي **~50MB** مع تقليم الأقدم تلقائياً.
- القاعدة **لكل جهاز** — لا تزامن بين الأجهزة في هذا النطاق (تصدير/استيراد يدوي مرشّح مستقبلي).

### الالتقاط على مرحلتين (حل تعارض «قيمة مبكرة vs تراكم بيانات»)
- **capture-lite من P1:** الأحداث التي يتوسطها التطبيق نفسه بلا حاجة لـOSC 133 —
  تشغيلات أوامر الكتالوج، قبول/رفض الاقتراحات، أفعال الدردشة. التراكم يبدأ من اليوم الأول.
- **الالتقاط الكامل في P3** (بعد OSC 133 في P2): كل الأوامر + exit codes في المخطط القائم.

### الاستفادة المحلية (تعمل بلا أي مزوّد ولا اتصال)
- **ترتيب اقتراحات محلي:** دالة تقييم حتمية صرفة (وزن×تكرار + اضمحلال حداثة) خلف مفتاح تعطيل.
- **جسر CommandCatalog** (الاقتراحات تسكن الأسطح الموجودة لا سطحاً جديداً):
  1. اقتراح «حفظ في الكتالوج» للقوالب المتكررة — عتبة **≥5 استخدامات في 30 يوماً** وغير موجود
     في الكتالوج (المطابقة بتطبيع مدخلات الكتالوج **بنفس المُطبِّع** ومقارنة hash)؛ يعرض أكثر
     متغيّر ملموس تكراراً مع خيار الصيغة المُعلمَنة؛ **اقتراح واحد لكل قالب** والرفض دائم.
  2. مزج ترتيب المشغّل بالتكرار/الحداثة (قابل للإيقاف من الإعدادات).
  3. «حفظ في الكتالوج» لناتج لغة طبيعية → أمر.

### نافذة «ذاكرة التطبيق» — الواجهة المرئية لقاعدة المعرفة (P3)
أربعة أقسام: أوامرك المتكررة / الأخطاء وحلولها المقبولة / **ملفك التعريفي كما سيُرسَل حرفياً**
(من نفس مسار ProfileBuilder المخبأ بإصداره — لا محاكاة موازية) / سجل الاقتراحات ونسبة القبول.
أفعال مقيّدة آمنة: حذف **متسلسل** (قالب ← أحداثه ← اقتراحاته)، حظر، تثبيت — لا تحرير نص حر
للإحصاءات. + عدّاد المعاينات القسرية + زر مسح كامل + تعطيل التسجيل كلياً.
القراءة عبر لقطات WAL والكتابة عبر طابور الكاتب نفسه. حالة فارغة تشرح ما سيتعلمه التطبيق.

### Profile Builder (P4)
«ملف معرفة المستخدم» مضغوط بسقف **~1500 توكن**، يُبنى على خيط خلفي عند خمول التطبيق،
مُرقَّم الإصدار، يُحقن في البادئة الثابتة للبرومبت. **أثره يُقاس** بنسبة قبول الاقتراحات قبل/بعد الحقن.

## 5) مراحل التنفيذ — 4 موجات إصدار

> كل موجة مكتملة = رفع `AppVersion.Current` (MINOR) + `ReleasedDate` + مدخلة `CHANGELOG.md`
> بالبنية القابلة للتحليل الآلي، بعنوان HIGHLIGHTS محسوس — بحكم دستور المشروع.

### P1 — «أول إجابة AI»
المحوّلان + أعلام قدرات دنيا + DPAPI (مع مسار إعادة الإدخال) + صفحة إعدادات AI + اختبار الاتصال
+ خادم SSE وهمي + دردشة دنيا (بثّ مُجمَّع + كتل مسيَّجة + نسخ + LTR) + خريطة الأخطاء
+ بطاقة أول تشغيل + اكتشاف Ollama عند الطلب + **إنشاء مخطط SQLite كاملاً + capture-lite**.
**DoD:** مستخدم جديد — بمفتاح واحد أو عبر Ollama بلا مفتاح — يصل لأول إجابة خلال < 3 دقائق.

**الترتيب الداخلي إن اضطر التقسيم (العمود الفقري أولاً):**
1. `IAiProvider` + `OpenAiCompatProvider` + بثّ SSE + اختبارات الخادم الوهمي (المخاطرة الأعلى
   تُستنزف أولاً؛ Ollama مسار تطوير بلا مفتاح).
2. مخطط SQLite كاملاً (ساعات قليلة — لا يؤجَّل).
3. إعدادات دنيا: مزوّد + مفتاح DPAPI + نموذج افتراضي + اختبار اتصال.
4. لوحة دردشة دنيا: بثّ خام + إلغاء + نواة خريطة الأخطاء (مفتاح خاطئ/شبكة/حد معدّل).
5. الكتل المسيَّجة + نسخ + LTR (بالعقد التزايدي).
6. `AnthropicProvider`.
7. بطاقة أول التشغيل + فحص Ollama.
8. capture-lite — آخر ما يُبنى لأنه غير مرئي، لكنه **لا يُقصّ** من الموجة.

### P2 — «التيرمنال يفهم»
OSC 133 في محرك VT + معالج الإعداد (diff حرفي + نسخة احتياطية + تأكيد صريح) + التدهور الرشيق
+ باني السياق (الميزانيات، إعادة الوصل، redaction، نموذجا الموافقة، المعاينة القسرية بضوابطها،
الشارة) + «اشرح هذا» / «أصلح آخر فاشل» + لغة طبيعية → أمر + رقاقة الخطأ (بصمات في ذاكرة
الجلسة) + الوضع الهادئ.
**DoD:** بعد أمر فاشل، نقرة واحدة تُنتج شرحاً للخطأ.

### P3 — «التطبيق يتعلّم»
الالتقاط الكامل + الترتيب المحلي في المشغّل + جسر CommandCatalog الثلاثي + نافذة «ذاكرة التطبيق»
+ ترقية بصمات الرقاقة إلى دائمة + **«رأيت هذا من قبل — الحل السابق»** من `error_patterns`.
**DoD:** المستخدم يرى ما تعلّمه التطبيق ويحذف بنداً منه، ويستلم اقتراح حفظ أمر متكرر، ويشاهد
حلاً سابقاً يُعرض عليه offline عند تكرار خطأ.

### P4 — «التكيّف والصقل»
Profile Builder والحقن + قياس نسبة القبول قبل/بعد + Markdown كامل + عدّادات استهلاك التوكنز
+ المزوّد المخصص + اختيار الموديلات + **backlog معلن:** حفظ المحادثات opt-in (معطل افتراضياً،
redaction قبل الكتابة، مشمول بالمسح الكامل).
**DoD:** البروفايل يظهر حرفياً في «ذاكرة التطبيق» ونسبة القبول تُقاس قبل وبعد إرفاقه.

## المخاطر المتبقية (من الاعتماد النهائي)

1. **خلط RTL/LTR:** يُختبر مبكراً بإجابة عربية تتضمن كتلة كود — أكثر موضع كسور بصرية ترجيحاً
   في WPF ولا يُكتشف بالإنجليزية وحدها.
2. **زحف نطاق P1:** الإغراء الأكبر صقل اللوحة؛ الحد المعلن (خام + كتل مسيَّجة فقط) يُحترم حرفياً.
3. **شذوذات SSE بين المزوّدين:** تُغطّى في الخادم الوهمي قبل إضافة كل مدخلة كتالوج، لا بعد الشكاوى.
4. **مسار DPAPI متعدد الأجهزة:** يُختبر فعلياً بنسخ ملف الإعدادات لجهاز ثانٍ.
5. **إجهاد المعاينة القسرية:** يُراقب عدّاد نسبتها؛ تجاوز ~30% = ضبط كاشف الإنتروبيا وallow-list.

## قرارات محسومة في المناظرة (خلاصة)

- محوّلان فقط (عام + Anthropic)؛ Gemini عبر نقطته المتوافقة — لا محوّل ثالث.
- لوحة الدردشة لا تعتمد OSC 133 → «أول إجابة AI» في الموجة الأولى؛ الالتقاط الكامل بعد OSC 133.
- المخطط الكامل + capture-lite من P1 = لا موجة «صامتة» ولا بيانات ضائعة.
- الفعل الصريح = موافقة على مقتطفه؛ المعاينة القسرية فقط عند حجب فعلي؛ السياق المحيط opt-in.
- لا دردشات على القرص الآن (نسخ يدوي متاح)؛ الحفظ opt-in مؤجَّل معلن في P4.
- Kimi وZ.ai مدخلتان مسمّاتان جاهزتان في الكتالوج — لا «مزوّد مخصص».
- redaction تخفيف لا ضمانة — خط الدفاع الحقيقي المعاينة + مقتطفات أضيق + لا متغيرات بيئة.
