## نظرة عامة على المشروع
بناء برنامج تيرمنال احترافي لويندوز:
- **اللغة:** C# / .NET 10
- **الواجهة:** WPF + CommunityToolkit.Mvvm
- **النظام المستهدف:** Windows 10 1903+ / Windows 11
- **الرندرنغ:** SkiaSharp (خلف واجهة `IRenderer` قابلة للاستبدال)
- **التخزين:** SQLite (ملف واحد) + JSON للإعدادات

## 📋 قواعد العمل (إلزامية)

0. **🔒 قفل الجلسة قبل أي عمل.** المشروع تعمل عليه جلسات/عملاء متوازية على شجرة مشتركة. في `PROGRESS.md` عمود «الجلسة»:
   - قبل بدء أي تاسك تحقّق أنّ خانة جلسته **فارغة**؛ إن كانت مملوءة بمعرّف جلسة أخرى فهو **محجوز — لا تعمل عليه** واختر غيره.
   - عند البدء: اكتب معرّف جلستك + بدّل الحالة إلى 🔄 + احفظ **فوراً قبل الشروع**.
   - عند الإكمال أو التوقّف: **امسح** خانة الجلسة (✅ عند الإكمال، ⏸️ + سبب عند التوقّف) حتى يتحرّر التاسك.
   - **ممنوع منعاً باتّاً** أن تعمل جلسة على تاسك جلسة أخرى قبل أن تُكمله وتُفرّغ خانته.
1. **تاسك واحد بكل مرة.** لا تبدأ تاسك جديد قبل إكمال الحالي بكل معايير القبول.
2. **احترم الترتيب والتبعيات.** لا تنفذ تاسك قبل إكمال كل التاسكات المذكورة بحقل "يعتمد على".
3. **لا تنتقل لمرحلة جديدة** قبل إكمال كل تاسكات P0 و P1 بالمرحلة الحالية.
4. **حدّث الحالة** بمكانين بعد كل خطوة:
   - checkbox الـ subtask داخل ملف المرحلة: `- [ ]` → `- [x]`
   - حالة التاسك برأس التاسك وبجدول `PROGRESS.md`: ⬜ لم يبدأ → 🔄 قيد العمل → ✅ مكتمل
5. **اكتب Unit Tests** لكل منطق بـ Terminal.Core (الـ Parser بالذات يحتاج تغطية عالية).
6. **معايير القبول شرط إغلاق.** لا تعلّم التاسك ✅ إلا إذا كل معايير القبول متحققة فعلياً ومجرّبة.
7. **الـ Commits:** رسالة بصيغة `T-XXX: وصف مختصر بالإنجليزي` بعد كل subtask مكتمل.
8. **إذا واجهت عائق تقني** (API غير متاح، تعارض تصميمي): وثّقه بقسم "ملاحظات التنفيذ" بنهاية التاسك واسأل المستخدم — لا تتجاوز بحل مؤقت صامت.
9. **لا تعدّل تاسكات مكتملة** بدون طلب صريح، وإذا تعديلك كسر شي قديم صلّحه ضمن نفس التاسك.
10. **الأمان:** لا تخزن أي كلمة مرور أو مفتاح Plain Text — استخدم DPAPI دائماً.
